Project

Inbesmnet.in

The current project on which I am working is based on the microservice architecture. We have MongoDB as a database for the DB caching. It has microservices deployed on EKS containers with Blue-Green deployment strategy. Each microservice API is written in JAVA. For high availability, each microservice is run with minimum 4 containers and the Application Load Balancer distrubutes the traffic among them internally. Afterwards there are 10 micro-frontends that are hosted in S3 in association with CloudFront. The Front-End is coded with Angular. The infrastructure of a project is managed with Terraform and uses the latest technology tools and services such as S3, CF, EKS, ALB, AWS Secretes Manger, MongoDB (SaaS), R53, SSL, IAM, VPC, WAF, CloudWatch, redis(aws service), SG, Lambda (only integrate did not write program), AWS Cognito. The tools selected for CI/CD are Git, Gitlab-ci, Maven, npm (for angular), SonarScan, Selenium, S3 & ECR. There is separate CI/CD pipelines used for backend & front-end. Branching Strategy: The developer team is responsible for managing the Develope Brnach. The developers merge their code from their respective Feature Branches into the Develope Branch. Its is then combines with the previous hotfixes and bug fixes and then the stable code get merged with the Release Branch. The code from the Release Branch is used in Test and UAT environment. If it get accepted, it is then merged with Main Branch for the Deployment in the Production Environment. CI/CD: The Developers commits their codes in the source code repositories in Github after the successfull local testing. The source code is then pulled from the repositories and sends for the inspection of vulnerabilities and code bugs. After testing the code, it moves for compilation and building the artifacts. Where the Continous Integration Environment builds and tests the artifacts. After the successfull testing of builds, it get published to the Artifact Repository that is AWS S3. The next stage is set for the Container Image building and publishing to the Container Registry. The CI/CD tool pulls the tested Artifacts from the Artifactory and builds the Container Images with the help of Dockerfile. The Container Images then pushed to the ECR. Then application get deployed on the Clusters through the EKS service. After the successfull deployment, the application is being monitored by AWS monitoring services. The project has environments like sandbox, Dev, Test, UAT and Prod.
